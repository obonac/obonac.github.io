# AyeHandsum Usage Guide

## Overview

AyeHandsum is a registry-driven symbolic encoding framework capable of losslessly transforming arbitrary digital data into reversible symbolic alphabets.

The repository currently contains predefined symbol registries including:

* Hands
* DNA / Genes
* Emoji
* Hieroglyphs
* Indus
* Bengali
* Telugu
* Research alphabets
* Toy alphabets
* Literary alphabets (Dickens, Wordsworth)

The encoding is reversible. Any digital file can be encoded into one of the supported alphabets and later decoded back into its original binary form.

---

# Runtime Requirements

AyeHandsum was designed to execute under **AyeVDI**.

Before using any command, activate the runtime:

```bash
ayesh=$(tempfile)
wget https://bit.ly/ayevdi-sfrom-init -O ${ayesh} -q
. ${ayesh}
```

This initializes the `sourcefrom` mechanism used by every command.

---

# Execution Model

The front-end commands are lightweight launchers.

Typical flow:

```
User

↓

ayehandsum

↓

sourcefrom()

↓

Download raw GitHub script

↓

Load registry

↓

Encode / Decode

↓

Output
```

Most functionality is executed directly from the GitHub repository using `sourcefrom`.

# Primary Commands

## Encode

```
ayehandsum <input-file> > encoded.txt
```

Internally invokes

```
ayesha encode hands
```

---

## Decode

```
ayehandsum-decode < encoded.txt > restored.bin
```

Internally invokes

```
ayesha decode hands
```

---

## Test

```
ayehandsum-test
```

Runs the Hands encoder test suite.

---

## Complete Test Suite

```
ayehandsum-test-all
```

Runs all predefined tests.

# Core Engine

The repository contains two generic engines.

## Encoder

```
ayecode-encode
```

Transforms canonical numeric symbols into the selected output alphabet.

---

## Decoder

```
ayecode-decode
```

Performs the reverse transformation.

---

# Symbol Registries

Each registry defines the alphabet used during encoding.

Examples:

```
ayecode-base-genes
ayecode-base-hands
ayecode-base-hiero
ayecode-base-anahiero
ayecode-base-indus
ayecode-base-bangla
ayecode-base-telugu
ayecode-base-sarega
ayecode-base-toy
ayecode-base-ayeun
ayecode-base-dickens
ayecode-base-wordsworth
ayecode-base-research
```

Changing the registry changes only the output alphabet.

The encoding engine remains unchanged.

# Configuration

Runtime parameters are generated by

```
ayecode-config
```

including:

* alphabet
* symbol count
* code width
* number of terms
* hash algorithm
* iteration count

# DNA Encoding

The DNA registry (`ayecode-base-genes`) maps encoded data onto a DNA-inspired alphabet.

Typical workflow:

```
Binary

↓

Encode

↓

DNA Symbols

↓

Store / Transfer

↓

Decode

↓

Original Binary
```

The repository demonstrates reversible encoding of arbitrary binary data into DNA symbols.

Biological synthesis constraints (GC balancing, ORFs, primers, sequencing error correction, etc.) are outside the scope of the current implementation.

---

# Registry Architecture

```
Input Data

↓

Canonical Numeric Representation

↓

Registry Selection

↓

Generic Encoder

↓

Symbol Stream
```

The decoder performs the inverse transformation using the same registry.

# Design Principles

* Lossless transformation
* Reversible encoding
* Registry-driven architecture
* Symbol-domain independence
* Generic encoder / decoder
* Extensible symbol registries

# Current Status

The symbolic encoding framework is functional.

Future work for biological DNA storage includes:

* FASTA generation
* GC-content optimization
* Homopolymer reduction
* Error-correcting codes
* Primer generation
* ORF-aware sequence generation
* Wet-lab validation
* Sequencing pipeline integration

These enhancements are independent of the core reversible encoding engine.
